# response-conventions

**何をするか**: Claude が調査結果・構造化レポートを提示するときの返答スタイルと出力フォーマット規約を提供する。

**起動**: 自動参照（`user-invocable: false`）。
調査結果の提示・複数案の比較・構造化レポートの作成前に Claude が自律的に読み込む。

---

## 適用タイミング

- 技術調査・検討結果をまとめるとき
- 複数の実装案を比較・提案するとき
- 構造化されたレポートやサマリーを書くとき

## 調査結果まとめの構成

| セクション | 内容 |
|---|---|
| 1. 要約 | 一文で結論を述べる |
| 2. 背景 | なぜ調査が必要だったか |
| 3. 結論 | 選択した案と一言理由 |
| 4. 調査結果 | 発見した事実・制約・前提 |
| 5. 修正案 | 比較表 + メリデメ + 修正内容 + 影響範囲 |
| 6. おすすめの案 | 推奨案と理由 |

## 修正案セクションのルール

- 案が複数ある場合は必ず**比較表**を作る
- メリット・デメリットを**両方**記載する
- 修正内容（ファイル・変更箇所・before→after）を具体的に示す
- 影響範囲（対象ファイル・破壊的変更の有無）を明示する
