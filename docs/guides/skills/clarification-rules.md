# clarification-rules

**何をするか**: Claude がリクエストを解釈する前に「立ち止まってユーザーに確認すべきか」を判断するルールを提供する。
スコープの境界・確認が必要な条件・安全に進めてよい条件を定義する。

**起動**: 自動参照（`user-invocable: false`）。
計画策定・曖昧なリクエストの解釈・変更対象の決定の前に Claude が自律的に読み込む。

---

## 主なルール

### 確認が必要な条件

- リクエストの対象・手段・目的地が不明確なとき
- 変更が複数ファイルに波及する可能性があるとき
- 削除・上書き・破壊的操作を行う前

### 安全に進んでよい条件

- 対象ファイルが明確に指定されている
- 変更内容が小さく、元に戻せる

### スコープ判断

選択肢の提示を「意図確認の代わり」にしない。
対象・手段・目的が不明確なときは、調査・実装の前に意図を確認する。
