# plain-doc

**何をするか**: 指定したファイルまたはテキストを読み込み、専門用語を平易な言葉で説明した Markdown リファレンスドキュメントを `docs/reference/` 配下に保存する。

**起動フレーズ**: `このファイルを分析して <パス>` / `まとめて` / `analyze this file` / `summarize this document`

---

## 使い方

```
このファイルを分析して ~/.claude/statusline.sh
```

出力先パスを省略した場合は、既存の `docs/reference/` 構造を確認したうえでパスを提案して確認を取ってから保存する。

## 出力先

```
docs/reference/<内容に即したパス>/overview.md
```

## 生成ドキュメントの特徴

- 専門用語・略語を平易な言葉で説明
- コードの「何をするか」を日本語で記述
- ソースファイルは変更しない

## 注意事項

- バイナリファイル（画像・PDF・コンパイル済みバイナリ）は対象外
- 出力言語はソースの主要言語に合わせる（日本語コメント → 日本語出力）
