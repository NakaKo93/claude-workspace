# 利用可能なスキル一覧

このワークスペース（`~/.claude/skills/`）に存在するスキルの使い方。

---

## plain-doc

**何をするか**: 指定したファイルまたはテキストを読み込み、専門用語を平易な言葉で説明した Markdown リファレンスドキュメントを生成して `docs/reference/` 配下に保存する。

**起動フレーズ**（どれでも可）:
- `このファイルを分析して <パス>`
- `このファイルをまとめて <パス>`
- `このファイルを説明して <パス>`
- `analyze this file <パス>`
- `summarize this file <パス>`

**入力**:
| 入力 | 渡し方 | 必須 |
|---|---|---|
| ソースファイルのパス | メッセージに書く | ○（またはインラインテキスト） |
| 出力先パス | メッセージに書く | △（省略時は確認あり） |

**出力先**: `docs/reference/<内容に即したパス>/overview.md`
→ 出力先を省略した場合、既存の `docs/reference/` 構造を確認したうえでパスを提案し、確認を取ってから保存する。

**処理の流れ**:
1. 入力解決・出力パス確認（メイン）
2. `plain-doc-analyzer` サブエージェントに委譲（ファイル読み込み・分析・執筆）
3. 保存確認をメインに返す

**制約**:
- 元のファイルは変更しない
- バイナリファイル（画像・PDF・コンパイル済みファイル）は対象外
- 出力言語はソースの主要言語に合わせる（日本語ソース → 日本語出力）

## reflection

**何をするか**: 直近の Claude Code セッションの JSONL ログを解析し、エージェント・スキル・フック・環境・プロセスの問題を特定して `fixes.json`（修正タスクリスト）を生成する。

**起動フレーズ**（明示的に指示したときのみ動作）:
- `反省して`
- `reflect`
- `reflect on today's session`
- `what went wrong? generate a fix list`

**入力**: なし（最新セッションの JSONL を自動検出）

**出力先**: `docs/tmp/reflection/artifacts/<セッションID>/`
- `reflection_input.json` — 抽出済みイベント
- `fixes.json` — 修正タスクリスト（優先度 P0/P1/P2 付き）

**処理の流れ**:
1. `~/.claude/projects/` 配下の最新 JSONL を検索
2. `extract_events.py` でイベント抽出・圧縮（元ファイルは変更しない）
3. イベントを分析して `fixes.json` を生成
4. サマリーを表示し、修正適用の確認を取る

**制約**:
- 自動起動しない（明示的な指示が必要）
- 同一 fingerprint のログは再解析をスキップ（重複防止）
- `thinking` ブロックの内部は解析しない

## skill-creator

**何をするか**: 新しいスキルの作成・既存スキルの改善をガイドする。初期化・バリデーション・パッケージングのスクリプトを提供する。

**起動フレーズ**:
- `スキルを作りたい`
- `このスキルを改善したい`
- `create a new skill`
- `build a skill for ...`

**主要スクリプト**（`skills/skill-creator/scripts/`）:

| スクリプト | 用途 |
|---|---|
| `init_skill.py <name> --path <dir>` | スキルディレクトリのテンプレートを生成 |
| `quick_validate.py <skill-dir>` | SKILL.md の構造・必須フィールドをチェック |
| `package_skill.py <skill-dir>` | バリデーション後に zip パッケージを生成 |

**スキルの置き場所**:
| スコープ | 場所 |
|---|---|
| 個人（全プロジェクト共通） | `~/.claude/skills/<name>/` |
| プロジェクト固有 | `.claude/skills/<name>/` |

## git-commit（commit）

**何をするか**: Git の差分を確認し、論理的な変更単位でコミットを分割する計画を立て、main 以外のブランチで作業する。

**起動フレーズ**:
- `/commit`
- `コミットして`
- `変更をコミットしたい`
